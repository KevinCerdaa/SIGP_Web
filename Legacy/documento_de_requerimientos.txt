2
Introducción
El proyecto consiste en desarrollar un sistema web para identificar, registrar y
analizar pandillas activas en la ciudad de San Luis Potosí. Esta plataforma busca ser una
herramienta efectiva para apoyar tanto a autoridades como a consultores y ciudadanos
en la detección y seguimiento de grupos pandilleriles mediante el registro estructurado
de información, geolocalización y análisis de actividades.
El sistema permitirá visualizar la ubicación geográfica de las pandillas mediante
longitud y latitud, así como consultar mediante su nivel de peligrosidad, los delitos o faltas
asociadas y sus antecedentes.
Además, contará con la capacidad de generar reportes detallados que faciliten la
toma de decisiones y el diseño de estrategias de prevención.
La plataforma contará con un esquema de roles diferenciados para controlar el
nivel de acceso y las funcionalidades. Dichos roles serán: Administrador, Consultor y
Ciudadano, cada uno con permisos específicos según sus responsabilidades dentro del
sistema.
Este sistema se desarrollará como una aplicación web moderna, adaptable a
diferentes dispositivos, y se integrará con servicios externos como Google Maps y
Telegram para mejorar la accesibilidad de la información.
Objetivo
Establecer explícitamente los requerimientos funcionales y no funcionales para el
desarrollo del sistema, definiendo el alcance, limitaciones y características esenciales
para su construcción.
El objetivo principal de este documento servirá como una guía detallada para los
alumnos desarrolladores y el docente evaluador, asegurando que todos los aspectos del
sistema estarán estrictamente detallados para su fácil entendimiento, desarrollo y
evaluación.
3
Alcance
El sistema permitirá registrar y consultar información de pandillas e integrantes
activos en San Luis Potosí, así como mostrar datos mediante mapas de Google Maps,
facilitar la generación de reportes detallados (delitos, faltas, peligrosidad, riñas, etc.), y
ofrecer consultas rápidas y accesibles mediante un bot de Telegram.
El sistema estará basado en un esquema de roles con diferentes niveles de acceso
y permisos sobre la plataforma y sus datos.
Roles
• Administrador: Registro, edición y eliminación de datos; generación de reportes;
acceso total mediante autenticación de login.
• Consultor: Consultas de datos, generación de reportes, cambio de contraseña y
vista general de datos; acceso mediante login.
• Ciudadano: Consulta pública de datos limitados, visualización de pandillas por
zona y nivel de peligrosidad sin necesidad de registro.
Funcionalidades principales
• Registrar pandillas e integrantes con sus datos generales, delitos y faltas
asociadas.
• Consultar información mediante filtrado de datos, búsqueda y mapas interactivos.
• Generar reportes descargables en formato PDF.
• Conexión con Google Maps para la consulta mediante ubicación (latitud y longitud)
y zonas de riesgo.
• Integración con un bot de Telegram desarrollado mediante BotFather y Python para
consultas rápidas.
• Desplegarse como una aplicación web responsiva usando Django y MySQL.
4
Descripción general del sistema
El sistema se desarrollará como una aplicación web bajo un sistema de capas
frontend, backend, una capa de datos y una capa de integración con servicios externos.
Arquitectura general
Frontend
La capa de presentación estará construida utilizando HTML, Tailwind CSS y
JavaScript, garantizando una interfaz moderna, intuitiva, responsiva y dinámica. Desde
esta capa, los usuarios podrán interactuar con el sistema a través de formularios, mapas
interactivos y menús de navegación adaptados a cada tipo de rol.
Backend
Será desarrollado con Python y su framework Django, encargado del
procesamiento de la información, la validación de datos, la autenticación de usuarios, la
gestión de roles, la generación de reportes y la comunicación con la base de datos.
Capa de datos
Se utilizará MySQL como sistema de gestión de base de datos relacional, donde
se almacenarán de forma estructurada la información de las pandillas, sus integrantes,
delitos y todo lo relacionado a ellas y a los distintos eventos, así como también los
usuarios del sistema.
Capa de integración
En la cual se incorporará la API de Google Maps, que permitirá representar
gráficamente las ubicaciones de las pandillas, sus zonas de actividad y niveles de
peligrosidad mediante marcadores interactivos. Asimismo, se desarrollará un bot de
Telegram usando el API de Telegram Bot y Python, que permitirá a los usuarios realizar
consultas rápidas sobre pandillas, zonas o niveles de riesgo, sin necesidad de acceder
directamente al sitio web.
5
Módulos
El sistema contará con distintos módulos funcionales:
• Módulo de registro y gestión de datos: Permitirá crear, editar y eliminar
información de pandillas e integrantes. Solo disponible para Administradores.
• Módulo de consulta geográfica: Desplegará información sobre zonas de riesgo
y ubicaciones en el mapa mediante pines filtrables.
• Módulo de reportes: Generará informes estructurados mediante JavaScript en
formato PDF sobre delitos, faltas, peligrosidad y eventos registrados.
• Módulo de seguridad: Gestionará el acceso mediante roles aplicando
autenticación y cifrado de contraseñas.
• Módulo de integración externa: Conectará el sistema con Google Maps,
Telegram y otros servicios web necesarios.
Finalmente, el sistema será desplegado en un entorno web accesible desde
cualquier navegador moderno y dispositivos móviles, garantizando disponibilidad
continua, compatibilidad multiplataforma y un desempeño óptimo en consultas y
generación de reportes.
Diseño preliminar de la base de datos
La base de datos implementada con MySQL almacenará toda la información
relativa a grupos pandilleros, sus integrantes, ubicaciones, delitos, faltas y usuarios del
sistema. Su objetivo es proporcionar una estructura ordenada y clara para facilitar su uso
tanto en el desarrollo como para los usuarios finales de la plataforma.
